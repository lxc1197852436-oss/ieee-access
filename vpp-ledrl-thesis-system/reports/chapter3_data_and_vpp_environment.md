# 第3章 数据体系与虚拟电厂环境建模

## 3.1 本章概述

虚拟电厂动态优化与决策问题依赖连续时间序列数据、可解释的物理约束以及能够反映外部事件影响的文本信息[1-3, 8]。与传统仅使用电价、负荷和新能源出力等数值型状态的调度模型不同，本文研究目标是构建融合中文文本事件与数值状态的语言增强深度强化学习框架[23-28]。因此，本章围绕“数据来源、数据处理、状态空间、动作空间、奖励函数和仿真环境”展开，形成后续 SAC 与 LE-DRL 算法训练的基础。

需要说明的是，国内真实虚拟电厂 15 分钟级运行数据和完整电力现货出清数据通常不公开。本文现阶段采用“国内公开披露数据校准 + 虚拟电厂仿真构造”的方式建立实验场景。公开数据用于校准研究背景、市场价格范围和新能源发展水平[15-18]；仿真数据用于构造强化学习所需的连续交互环境[21, 29, 46]。该处理方式能够保证实验可复现，同时避免将公开汇总数据误表述为真实运行明细。

## 3.2 国内公开数据来源

本文当前已整理三类国内公开数据，分别用于研究背景、参数校准和市场场景构建。

### 3.2.1 电力需求与发电规模数据

国家能源局发布的 2024 年全社会用电量数据用于说明中国电力需求增长背景[15]。已结构化文件为：

`data/processed/nea_2024_power_consumption.csv`

该文件包含全社会用电量、规模以上工业发电量、第一产业用电量、第二产业用电量、第三产业用电量和城乡居民生活用电量等指标。根据已抓取数据，2024 年全社会用电量为 98521 亿千瓦时，同比增长 6.8%。该类数据主要用于论文绪论和实验场景规模合理性说明，不直接作为强化学习逐时输入。

### 3.2.2 可再生能源装机与发电数据

国家能源局发布的 2024 年可再生能源并网运行情况用于说明新能源高比例接入背景[15]。已结构化文件为：

`data/processed/nea_2024_renewable_summary.csv`

该文件包含可再生能源新增装机、累计装机、水电、风电、太阳能、生物质发电装机以及可再生能源发电量等指标。已抓取数据显示，2024 年全国可再生能源装机达到 18.89 亿千瓦，其中风电装机 5.21 亿千瓦，太阳能发电装机 8.87 亿千瓦。该类数据用于支撑光伏和风电在虚拟电厂中的重要性。

### 3.2.3 广东电力现货与独立储能公开披露数据

广东电力现货市场与独立储能相关公开披露信息用于校准实验中的价格范围和储能套利逻辑。储能参与市场套利和辅助服务的合理性可由储能调度与电池储能综述研究支撑[31, 44, 45]。已结构化文件为：

`data/processed/guangdong_market_storage_2024.csv`

该文件包含广东现货日前价格波动范围、独立储能充放电电量、充放电均价、充放电价差、市场主体规模等指标。当前样例场景将广东披露的现货日前价格区间校准为 68 元/MWh 至 525 元/MWh，并将独立储能价差作为储能商业模式合理性依据。

## 3.3 数据表结构与字段说明

本文强化学习环境采用统一的 15 分钟级 VPP 调度数据表。当前样例数据文件为：

`data/processed/china_vpp_priority1_guangdong_sample.csv`

数据覆盖 2025 年 7 月 1 日至 2025 年 7 月 30 日，共 2880 行。字段如下。

| 字段 | 单位 | 含义 | 是否入模型 |
|---|---:|---|---:|
| `timestamp` | - | 15 分钟调度时刻 | 是 |
| `region` | - | 区域，当前为广东省 | 否 |
| `load_mw` | MW | 虚拟电厂聚合负荷 | 是 |
| `pv_mw` | MW | 分布式光伏出力 | 是 |
| `price_yuan_mwh` | 元/MWh | 电力市场价格 | 是 |
| `temperature_c` | 摄氏度 | 环境温度 | 是 |
| `event_type` | - | 文本事件类别 | 是 |
| `event_text` | - | 中文文本事件内容 | 是 |
| `source_note` | - | 数据边界说明 | 否 |

文本事件包括正常运行、高温预警、需求响应、价格尖峰和新能源消纳五类。它们模拟虚拟电厂在真实运行中可能收到的气象预警、市场公告和调度提示，为语言增强状态提供输入。

## 3.4 数据处理流程

数据处理流程分为五步：

1. 公开数据抓取：通过脚本抓取国家能源局、广东市场披露等网页，并保存原始 HTML 和文本。
2. 结构化抽取：使用正则规则抽取年度用电量、可再生能源装机、广东现货价格范围和储能相关指标。
3. 场景参数校准：用广东公开披露的价格范围校准仿真价格序列，用国家能源局数据说明新能源和负荷背景。
4. 15 分钟场景生成：构造夏季负荷曲线、光伏出力曲线、温度曲线、电价曲线和文本事件。
5. 数据质量校验：检查字段完整性、缺失值、时间戳连续性、数值范围和环境可运行性。

当前校验结果显示：样例数据共 2880 行，无必需字段缺失，无重复时间戳，无不规则 15 分钟步长，数值字段无缺失值。事件分布为：正常运行 2767 条，高温预警 42 条，价格尖峰 41 条，新能源消纳 17 条，需求响应 13 条。

## 3.5 虚拟电厂系统结构

本文考虑的虚拟电厂由分布式光伏、储能系统、聚合负荷和电力市场接口组成[1-3, 14]。虚拟电厂在每个调度步根据当前负荷、光伏、电价、温度、储能 SOC 和文本事件选择储能充放电动作，从而实现经济收益最大化和风险控制。

系统功率平衡可表示为：

```text
P_net,t = P_pv,t - P_load,t + P_bat,t
```

其中，`P_net,t > 0` 表示虚拟电厂向电网售电，`P_net,t < 0` 表示虚拟电厂从电网购电。`P_bat,t > 0` 表示储能放电，`P_bat,t < 0` 表示储能充电。

## 3.6 马尔可夫决策过程建模

本文将虚拟电厂调度问题建模为马尔可夫决策过程，该建模方式在电力系统强化学习和储能控制研究中较为常见[9, 21, 29, C1]：

```text
M = <S, A, P, R, gamma>
```

其中，`S` 为状态空间，`A` 为动作空间，`P` 为状态转移概率，`R` 为奖励函数，`gamma` 为折扣因子。

### 3.6.1 状态空间

数值状态定义为：

```text
s_num,t = [hour_t, load_t, pv_t, price_t, temperature_t, SOC_t]
```

语义状态由文本事件编码得到：

```text
s_sem,t = Encoder(event_text_t)
```

当前系统中，本地语义编码器输出风险分数、价格尖峰分数、负荷压力分数和新能源消纳压力分数。后续接入主流大模型或中文预训练模型后，可将文本编码替换为更高维的语义向量。

增强状态定义为：

```text
s_aug,t = [s_num,t, s_sem,t]
```

标准 SAC 基线仅使用 `s_num,t`，LE-DRL 使用 `s_aug,t`。

### 3.6.2 动作空间

动作为空间储能充放电功率：

```text
a_t = P_bat,t
```

其中：

```text
-P_bat_max <= P_bat,t <= P_bat_max
```

当前环境中，储能最大功率为 2 MW，储能容量为 8 MWh。正值表示放电，负值表示充电。

### 3.6.3 储能状态转移

设调度步长为 `Delta t`，储能容量为 `E_cap`，充电效率为 `eta_ch`，放电效率为 `eta_dis`。当储能充电时：

```text
SOC_{t+1} = SOC_t + |P_bat,t| * Delta t * eta_ch / E_cap
```

当储能放电时：

```text
SOC_{t+1} = SOC_t - P_bat,t * Delta t / (eta_dis * E_cap)
```

同时满足：

```text
SOC_min <= SOC_t <= SOC_max
```

当前环境设置 `SOC_min = 0.1`，`SOC_max = 0.9`，充放电效率均为 0.95。

### 3.6.4 奖励函数

奖励函数由四部分组成：购售电收益、储能退化成本、弃光惩罚和动作不可执行惩罚：

```text
r_t = Revenue_t - C_deg,t - C_curt,t - C_violate,t
```

其中：

```text
Revenue_t = P_net,t * Delta t * price_t
```

```text
C_deg,t = |P_bat,t| * Delta t * c_deg
```

由于省级新能源消纳率明细数据不稳定，本文不把外部消纳率表作为强化学习逐时输入，而是用环境内部的剩余光伏功率刻画即时消纳压力。国家能源局公布的全国风电平均利用率和光伏发电利用率用于背景校准和合理性说明。弃光惩罚用于约束光伏出力大于负荷且储能未能充分吸纳的情况：

```text
C_curt,t = max(0, P_pv,t - P_load,t - max(0, -P_bat,t)) * Delta t * c_curt
```

当智能体请求的动作因 SOC 边界无法执行时，环境会记录动作裁剪并施加惩罚：

```text
C_violate,t = c_violate, if P_request,t != P_actual,t
```

该奖励函数能够同时体现虚拟电厂的经济性、储能寿命和新能源消纳目标。

## 3.7 文本事件与语义状态构建

本文文本事件分为四类风险场景：

| 事件类型 | 语义含义 | 预期调度影响 |
|---|---|---|
| 高温预警 | 负荷上升、晚高峰压力增大 | 提前保留储能，避免过早放电 |
| 需求响应 | 晚高峰削峰需求增强 | 提升晚高峰放电或降低负荷 |
| 价格尖峰 | 市场价格上行风险 | 高价时段放电，提高套利收益 |
| 新能源消纳 | 午间光伏消纳压力 | 低价或高光伏时段充电 |

当前本地语义编码器基于关键词和规则输出语义风险分数。该模块用于保证没有 API key 时系统仍可运行。后续接入主流 AI 后，可通过统一接口将文本事件输入大模型，输出风险等级、价格趋势、建议动作和自然语言解释。

当前环境校验结果显示，前 96 个调度步中 SOC 范围为 0.3536 至 0.8325，未出现动作裁剪，说明储能约束和动作执行逻辑在样例数据上能够稳定运行。

## 3.9 本章小结

本章构建了面向中国虚拟电厂场景的数据体系和仿真环境。首先，基于国家能源局和广东电力市场公开披露数据建立研究背景与价格校准依据；其次，构造 15 分钟级广东虚拟电厂样例场景，包含负荷、光伏、电价、温度和中文文本事件；最后，将虚拟电厂调度问题建模为包含语义增强状态的马尔可夫决策过程，并设计了考虑经济收益、储能退化、弃光惩罚和动作约束的奖励函数。该环境为后续 SAC 与 LE-DRL 算法设计和实验对比奠定了基础。
